Purpose

The X-CUBE-AWS (STM32 AWS IoT software expansion for STM32Cube) provides ready-to-run firmware examples to support quick evaluation and development of AWS IoT Cloud applications on B-L475E-IOT01A, 32F413HDISCOVERY, 32F769IDISCOVERY and 32L496GDISCOVERY with cellular add-on board.

  • Three IDE are supported
    • IAR EWARM
    • Keil MDK-ARM
    • System Workbench
  • X-CUBE-AWS natively runs on four platforms
    • B-L475E-IOT01A and 32F413HDISCOVERY support Wi-Fi™ connectivity with an on-board Inventek ISM43362 module.
    • 32F769IDISCOVERY provides a native Ethernet interface.
    • P-L496G-CELL02, packed with the LTE Quectel BG96 cellular modem daughter board, supports cellular connectivity.
  • List of references to user documents
    • UM2178 - Getting started with X-CUBE-AWS Amazon web services IoT software expansion for STM32Cube
    • UM2426 - Getting started with the X-CUBE-CELLULAR cellular connectivity Expansion Package for STM32Cube
    • UM2262 - Getting started with the X-CUBE-SBSFU STM32Cube Expansion Package

Known Limitations

  • The P-L496G-CELL01 (Quectel UG96 3G modem) board is not specifically supported by the present expansion package anymore. However, it can be enabled by the user thanks to the cellular stack which is pre-integrated. Please refer to the documentation of X-CUBE-CELLULAR.

  • Network connection instability on B-L475E-IOT01A and 32F413HDISCOVERY:
    • The board may not connect to the WiFi access point at first trial.
    • Remote server connection failure or disconnection reported as a system error by the WiFi module.
  • System Workbench for STM32.
    • B-L475E-IOT01A board rev B and rev C requires to use System Workbench with version greater or equal to 2.1. A debug option must be changed:
      • Right click on the project, select “Debus as…”, “Debug Configuration…”, the “B-L475E-IOT01A.elf” target, then the “Debugger” tab
      • If the “Reset Mode” option is not visible, click on the “Show generator options…” button
      • The “Reset Mode” option must be set to “Software system reset”
    • The debugger starts the execution from the application main(), and not from the BootLoader main(). A workaround is to perform a hard-reset on the board to force a full reboot.
  • LwIP Middleware (32F769IDISCOVERY only): Disabled a compilation warning.
    • IAR
    - Pa182: bitwise operation drops significant bits from a constant

  • When building the Bootloader, some warnings appear linked to the disabled security features.

  • Recommendation when using the sw4stm32 debug button and if a build is required:
    • The build message “Generating the global elf file SBSFU and userApp” must be printed before the flash loader launch.
    • This is worth checking to be sure that the updated boot loader is used.

Main Changes

Prerequisite

  • Please install STM32CubeProgrammer from Version 2.0.0 to the proposed default location.

New features

  • Cellular connection on STM32L496G-Discovery with the board extension based on BG96 Quectel modem component and the cellular stack of the X-CUBE-CELLULAR package.
  • STM32_Connect_Library providing a flexible network abstraction.
  • Support of Secure Firmware Update, based on a bootloader derived from the X-CUBE-SBSFU package:
    • Generation of encrypted firmware update image.
    • Ensure authenticity and integrity of firmware update process.
    • Dual image support enable rollback and version check mechanism.
    • Asymmetric and symmetric cryptographic schemes supported.
    • IDE projects (SBSFU, SE_CoreBIN) to configure the bootloader.
    • Advanced elf generation to create a single executable image with bootloader and AWS application.
  • AWS jobs to remotely control user LED state from the AWS IoT console, or to trig a remote firmware update.
  • ST AWS dashboard.
  • Update to the last versions of HAL, BSP, “AWS IoT device SDK for embedded C”, FreeRTOS…

Bug fixes

  • eS-WiFi driver (B-L475E-IOT01A, 32F413HDISCOVERY): Driver improvements.
  • eS-WiFi RTOS support.
  • Configuration of the RTC frequency scalers on 32F413HDISCOVERY and 32F769IDISCOVERY.
  • Improved robustness to temporary network failures.

Maintenance release

Updated dependencies (see Contents list below)

Bug fixes

  • eS-WiFi driver (B-L475E-IOT01A, 32F413H-DISCO): Solved random network disconnections that were due to a de-synchronization between the SPI and the GPIO.

  • TCP/IP configuration (32F769I-DISCO)
    • Generate a unique MAC address derived from the CPU ID in order to avoid address collisions when several F7 boards are connected to the same LAN.
    • Activate the local port number true randomization, to avoid possible TLS session confusions on server side after a board reset.
    • Activate the TCP keep-alive. Otherwise, the remote host may disconnect earlier than the first MQTT keep-alive message.

  • Improved error management

Other changes

  • Introduction of an internal network socket abstraction layer: NET API.

  • Source tree update to isolate the files which are common to all Cloud Cube packages.

  • RFU
    • Complies to HTTP 1.1.
    • Inter-operable with the Apache web server.
    • Supports server-side connection close, and client reconnection.
    • Supports HTTPS (the root CA certificate must be provisioned together with the Cloud and time servers’).
  • RTC initialization from HTTP
    • Network load reduction: HEAD command instead of GET .
    • Supports HTTPS (the root CA certificate must be provisioned together with the Cloud and RFU servers’. See the application readme.txt for more details).
